# ADR-0103: Sampled Audio: Track Playback and Instrument Banks

- **Status:** Proposed
- **Date:** 2026-06-11

## Context

ADR-0039 committed the audio capability to MIDI event semantics: components mail `aether.audio.note_on` / `note_off`, the substrate synthesises voices from a static built-in registry, and everything the cap plays is generated by a hand-rolled kernel. That registry has grown from five oscillator patches to eleven (partial-bank piano family, noise/sweep percussion) without a wire change, which validates the append-only id scheme — and also marks the ceiling of the approach: every timbre must be expressible as a kernel the substrate hand-rolls, and there is no way to play audio that already exists as a file (a music track, ambience, a recorded sample set).

Two issues extend the surface past that ceiling:

- **Track playback** (iamacoffeepot/aether#1678) — play a decoded audio asset through the cap, addressed the way the rest of the substrate addresses files (fs namespace + path, ADR-0041).
- **Sampled instrument banks** (iamacoffeepot/aether#1679) — load a bank of pitched samples at runtime, append it to the instrument registry past the built-ins, and play it through the unchanged `note_on` / `note_off` surface.

Both need the same core the cap does not have today: fetch file bytes, decode them to PCM, and resample to the device rate. And the second forces a question ADR-0039 explicitly parked: what happens when the instrument registry stops being a static slice.

Forces carried over from ADR-0039 that still bind:

1. **Dumb substrate / composable user-space.** No bus model, no effects, no per-source mixing policy beyond what a kind field states explicitly.
2. **Desktop-only.** Chassis without an audio device reply `Err` to anything that has a reply, and nop the fire-and-forget kinds.
3. **The realtime callback cannot block.** Everything that reaches the cpal thread rides the existing lock-free event queue; decode and file I/O happen elsewhere.
4. **Single namespace authority.** ADR-0041 made the fs capability the one resolver of `namespace://path`. The audio cap must not grow a second copy of that registry.

## Decision

The audio capability gains a decode/resample core and two consumers of it: a track lane for whole-file playback, and a sampled-instrument loader that appends banks to a now-extensible registry. Files reach the cap as mail — the cap forwards `aether.fs.read` to the fs capability and correlates the reply — so namespace resolution stays single-sourced.

### 1. Decode/resample core

WAV is the v1 container (decoded with `hound`; 16-bit integer and 32-bit float PCM arms). Decode and resample run off the dispatcher via the ADR-0093 blocking-dispatch surface, never in a handler body and never on the audio callback. The core resamples to the device rate at load time with linear interpolation and downmixes to mono, so the callback-side hot path is an index walk over a flat `f32` buffer — the same shape the synth voices already have. Stereo persistence and compressed containers (ogg/flac/mp3 via a heavier decode dependency) are parked; the entry point dispatches on file extension so they slot in later without a wire change.

### 2. File fetch rides mail

`on_play_track` / `on_load_instrument` forward `aether.fs.read` to the `aether.fs` mailbox and park the request in a pending table keyed by the `(namespace, path)` pair the fs reply kinds already echo for exactly this correlation. A new `read_result` handler on the audio cap routes the bytes to the decode dispatch and, on completion, replies to the original requester. A bank load fans out to one read per referenced sample file and assembles when the last reply lands. The cap holds the original sender's reply route across the round-trip; failures at any stage (fs error, malformed file, unsupported format) come back as the `Err` arm of the requester's reply kind — loud, not logged-and-dropped, because a typo'd path is the common agent failure.

### 3. Track lane

Three kinds join the family, schema-shaped like the fs kinds (they carry strings):

- `aether.audio.play_track { namespace: String, path: String, gain: f32, looping: bool }` — replies `aether.audio.play_track_result`: `Ok { namespace, path }` / `Err { namespace, path, error }`.
- `aether.audio.stop_track { namespace: String, path: String }` — fire-and-forget; stopping a track that isn't playing is a no-op, matching `note_off`.

Tracks mix in their own lane: the synth holds a track list beside the voice pool, and a playing track neither counts against `MAX_VOICES` nor participates in voice-steal — a music bed must not be evicted by a note flurry. Tracks are keyed by `(sender_mailbox, namespace, path)`, mirroring the voice key, so one component cannot stop another's track and re-playing the same key restarts it. `stop_track` releases through a short (~5 millisecond) linear fade rather than truncating, to avoid the click. Per-track `gain` is a kind field set at play time, not a mixer surface — running-gain automation stays user-space per ADR-0039.

### 4. The instrument registry goes extensible

The static `BUILTINS` slice stays exactly as it is — ids 0 through 10 remain wire-stable, append-only, compiled in. Loaded banks append after it: the cap assigns ids from `BUILTINS.len()` upward in load order and ships each assembled bank to the callback thread as a registry event on the existing queue; the synth keeps a bank table it consults when an id walks past the built-ins.

**Loaded ids are session-scoped.** A loaded instrument's id depends on load order and does not survive a restart. The loading agent learns the id from the load reply and threads it through its own `note_on`s. The parked `resolve_instrument` name-lookup kind (ADR-0039 follow-up) is the eventual stable-name contract; it stays parked until something needs cross-session id stability, because the loader holding the reply already knows its id.

- `aether.audio.load_instrument { namespace: String, path: String }` — replies `aether.audio.load_instrument_result`: `Ok { instrument_id, name }` / `Err { namespace, path, error }`.

No unload and no dedup in v1: loading the same bank twice appends two ids, and banks live until the process exits. The load reply and the log line report the bank's resident bytes so an agent can see what it is spending.

### 5. SFZ subset as the bank format

A bank is an `.sfz` file plus the WAV files it references, fetched from the same fs namespace relative to the `.sfz` path. SFZ is chosen because freely licensed sample sets ship in it — the alternative is inventing a manifest and converting every set by hand. The subset is deliberately small, parsed by a hand-rolled line-oriented parser (the format is `opcode=value` runs under `<header>` tags; no dependency earns its place):

- Headers: `<control>` (for `default_path`), `<group>`, `<region>`, with group→region opcode inheritance.
- Opcodes: `sample`, `lokey` / `hikey`, `pitch_keycenter`, `lovel` / `hivel`.
- Unknown opcodes warn and are ignored, so real-world files load rather than failing on the first envelope opcode outside the subset.

The first delivery is piano-first: full-decay sets need no sustain loops, so the loop opcodes (`loop_start` / `loop_end` / `loop_mode`) are deliberately outside the initial subset and join with iamacoffeepot/aether#1682 — until then they fall under warn-and-ignore like any other opcode. Loop points arrive in source-sample frames and are scaled by the resample ratio at load time (the bank stores device-rate positions, fractional, so the wrap interpolates sub-sample).

### 6. Sample voice kernel

A third voice kernel joins `Oscillator` and `PartialBank`: region selected by `(pitch, velocity)`, played back at rate ratio `2^((pitch − pitch_keycenter) / 12)` with linear interpolation over the region's device-rate PCM, wrapped in the same short attack ramp / `note_off`-release shape the partial bank uses. The first delivery plays unlooped full-decay samples — a held note ends when its sample runs out, which is correct for the piano-first sets. Sustain loops (#1682) extend the kernel: while a region's loop is active the position wraps from `loop_end` back to `loop_start` (interpolating across the seam), holding the note indefinitely until `note_off` hands it to the release ramp; `loop_sustain` is approximated as `loop_continuous` under the release ramp, exact tail playback parked. Sample voices are ordinary voices: they count against `MAX_VOICES` and participate in voice-steal. The voice pool's element type stops being `Copy` (a sample voice holds a reference-counted bank handle); the pool was never structurally dependent on `Copy`.

### 7. Chassis policy unchanged

Desktop only. Headless and hub reply `Err` to `play_track` and `load_instrument` (they have replies — fail fast per ADR-0035), and nop `stop_track`. A desktop substrate whose pipeline fell back to nop (no device, `AETHER_AUDIO_DISABLE=1`) does the same.

## Consequences

### Positive

- **Music and ambience become assets, not kernels.** A Claude-authored game writes a WAV into a namespace with `aether.fs.write` (or ships one under `assets`) and plays it with one kind — no more encoding every sound as synthesis parameters.
- **Real instrument timbres without shipping them.** The SFZ subset makes existing freely licensed sample sets loadable as-is; the substrate binary stays sample-free.
- **The decode core is shared.** Track playback and bank loading exercise the same fetch → decode → resample path, so #1679 is mostly parser + registry + kernel on top of #1678.
- **`note_on` is the whole performance surface.** Sampled instruments are addressed exactly like built-ins; nothing upstream of the registry changes.

### Negative

- **Memory is unbounded in v1.** No bank unload, no track cache eviction — a session that loads ten pianos keeps ten pianos resident. Reported bytes in the load reply is observability, not a budget. An unload kind is the obvious follow-up when it bites.
- **Loaded ids don't survive restart.** Any persisted composition that references a loaded id must re-load and re-map at boot. `resolve_instrument` lifts this when it lands.
- **Mono only.** Stereo sets and stereo music beds collapse to mono until the lane grows channel support.
- **Linear interpolation is audible at extremes.** Repitching a sample more than a fifth or so from its root softens transients; dense multisampled sets mask this, single-sample banks won't.

### Neutral

- **The event queue grows non-`Copy` variants.** Registry and track events carry reference-counted PCM; the queue never required `Copy`.
- **Kind vocabulary grows by five.** Schema-shaped kinds, same family, no structural change to the manifest.

## Alternatives considered

- **Audio cap reads files directly from disk.** Rejected: duplicates the fs capability's namespace registry and path-safety policy; ADR-0041 has one resolver and mail reaches it.
- **Streaming decode for tracks.** Rejected for v1: whole-file decode of a three-minute mono track at 48 kilohertz is roughly 35 megabytes — acceptable; a refill protocol touching the realtime thread is the riskiest kind of code in this subsystem.
- **SoundFont (`.sf2`) as the bank format.** Re-rejected (ADR-0039 already declined it): binary blob format, heavier parse, and the licensing on free soundfonts is murkier than per-sample sets.
- **A full SFZ engine dependency.** Rejected: the crates are immature, and the subset that covers real sample sets is small enough that a strict hand-rolled parser with warn-and-ignore unknown opcodes is less total risk.
- **Runtime-defined synthesis patches instead of samples.** Orthogonal, still parked: patches extend what the synth can generate; banks play what someone already recorded. Neither substitutes for the other.
- **Raw PCM kinds on the wire.** Re-rejected (ADR-0039): components mailing sample buffers is the wrong altitude; files ride the fs surface, references ride mail.
- **Tracks as voices (count against `MAX_VOICES`).** Rejected: voice-steal evicting the music bed to admit the 65th note is exactly the wrong trade; the lanes have different lifetimes and different steal policies (tracks are never stolen).

## Follow-up work

- **#1678**: decode/resample core, fs-mail fetch + pending-table correlation, track lane, `play_track` / `stop_track` / `play_track_result` kinds.
- **#1679**: SFZ subset parser (piano-first, unlooped), extensible registry + `load_instrument` kinds, sample voice kernel.
- **#1682**: sustain loops — `loop_start` / `loop_end` / `loop_mode` opcodes, loop-region cycling in the sample voice; unlocks the sustaining sets (organ, bowed strings).
- **Parked**: bank unload / track cache eviction; stereo lane; compressed containers (ogg/flac); `resolve_instrument` for cross-session id stability; exact `loop_sustain` tail playback; streaming decode for long-form audio.
